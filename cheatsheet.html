第1章 網路基礎入門

🔑 OSI 七層（上→下）

  • 7應用層 HTTP/DNS/SSH｜6表示層 加密/格式｜5會話層 建立/終止連線
  • 4傳輸層 TCP/UDP｜3網路層 IP/路由器｜2資料鏈路層 MAC/ARP/交換機｜1實體層 位元流/線纜
  • ISO 於 1984 提出；TCP/IP 為 4-5 層實用模型

🔑 TCP vs UDP

  • TCP：連接導向、可靠、有序、有三向交握；UDP：無連接、快、無重傳
  • IP 特性：無連接、盡力而為(Best Effort)、不可靠

🔑 通訊埠

範圍類別
0–1023熟知埠
1024–49151註冊埠
49152–65535動態/私有
  • HTTP 80／HTTPS 443／SSH 22／DNS 53／FTP 21控制,20資料／SMTP 25
  • 總範圍 0–65535（16 位元）

Wireshark

  • 顯示過濾：http.request.method=="GET"（越精確越好）
  • 擷取過濾(BPF)：host x / port 80（越寬鬆越好、要謹慎）
  • 埠掃描工具：Nmap、Masscan

第2章 系統基礎入門

🔑 Linux 權限

  • r=4 w=2 x=1；755=rwxr-xr-x；644=rw-r--r--
  • 對象：擁有者u/群組g/其他o；指令 chmod/chown/chgrp/umask
  • /etc/shadow=加密密碼、只有 root 可讀

🔑 特殊權限（易考提權）

  • SUID(4)：以擁有者權限執行 → root+SUID 易被提權
  • SGID(2)：以群組權限執行；Sticky(1)：限制刪除（/tmp）

常用指令

  • ls/cd/pwd/cp/mv/rm；rm -rf / 破壞性→--preserve-root
  • tail -f 即時看日誌；grep 找可疑；ps aux 找可疑程序
  • whoami/id/su/sudo；管道 |；重定向 >覆蓋 >>追加

Shell 安全

  • PATH 劫持：竄改 PATH 使惡意程式優先執行
  • 命令注入、勿貼不明指令、.bashrc 風險

第3章 網站開發基礎

HTTP

  • GET（參數在 URL、有長度限制）／POST（資料在 body、送表單）
  • 狀態碼：200 成功｜301/302 轉址｜401 需授權｜403 禁止｜404 找不到｜500 伺服器錯
  • HTTP 無狀態 → 用 Cookie/Session 維持登入

前端三本柱

  • HTML 結構／CSS 樣式／JavaScript 互動行為

🔑 常見網站攻擊

  • XSS：注入惡意 JS 在他人瀏覽器執行 → 防：輸出編碼、CSP
  • 會話劫持：竊取 Session ID
  • CSRF：誘導受害者送出非本意請求

第5章 資料庫基礎

🔑 SQL 四大操作 (CRUD)

  • SELECT ... FROM ... WHERE 查詢／過濾
  • INSERT 新增／UPDATE 修改／DELETE 刪列
  • DROP TABLE 刪表；TRUNCATE 清空

子句順序與用途

  • WHERE 過濾列 → GROUP BY 分組 → HAVING 過濾組 → ORDER BY 排序

鍵與關聯

  • 主鍵 PK：唯一、不可重複、不可 NULL、一表一個
  • 外鍵 FK：關聯另一表主鍵
  • JOIN 合併兩表；UNION 上下疊加（欄位數需相同→union 注入基礎）

第6章 資料庫安全入門

🔑 SQL Injection 原理

  • 未過濾輸入被當 SQL 語法。' OR '1'='1 使條件恆真 → 繞過登入
  • 註解符號 --+（--+空格）截斷後段語法

🔑 sqli-labs 攻擊流程（Less-1）

  • ?id=1' 觸發錯誤訊息（error-based）
  • order by N--+ 逐一試，找出欄位數（報錯前的最大值）
  • union select 1,2,3--+ 用 union 取出資料
  • 盲注：布林盲注（真假差異）／時間盲注（延遲）

🔑 防禦

  • 參數化查詢 / Prepared Statement（最根本）
  • 最小權限 DB 帳號、輸入驗證、關閉詳細錯誤訊息、WAF
