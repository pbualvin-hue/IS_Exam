🎯 大觀念（貫穿全部，先懂這個）

🔑 CIA 三大支柱

  • 機密性 Confidentiality：不被未授權者看到 → 靠加密、存取控制
  • 完整性 Integrity：不被竄改 → 靠雜湊/數位簽章
  • 可用性 Availability：需要時可用 → 靠備份、防 DDoS、負載平衡
  • 看任何攻擊都問「它傷哪個 C/I/A」：竊資料=C、竄改=I、DDoS/勒索=A

🔑 認證 vs 授權（超常混）

  • 認證 Authentication＝你是誰（登入）→ 失敗回 401，對 OWASP A07
  • 授權 Authorization＝你能做什麼（權限）→ 拒絕回 403，對 OWASP A01

🔑 三個防禦原則

  • 最小權限：只給必要權限，縮小「爆炸半徑」（Linux帳號/DB帳號/檔案權限都是）
  • 縱深防禦：多層互補，一層破了還有下一層（沒有銀彈）
  • 零信任：預設不信任，每次存取都驗證

🔑 兩組關鍵對比

  • 防止發生 vs 限制後果：參數化查詢/輸出編碼(防發生) ＋ 最小權限(限後果)
  • 偵測 vs 防禦：IDS/WAF監控(看見) → IPS/WAF主動(阻擋)
  • 注入類漏洞共通根因＝「資料與程式碼沒分離」（SQLi/XSS/命令注入同源）

🔐 加密・雜湊・密碼

🔑 對稱 vs 非對稱加密

  • 對稱（AES）：同一把金鑰、快，但要先安全交換金鑰
  • 非對稱（RSA）：公鑰加密/私鑰解密，解決金鑰交換與身分驗證
  • HTTPS：先用非對稱交換一把臨時金鑰，再用對稱快速傳資料

🔑 雜湊 vs 加密（易考）

  • 雜湊（SHA-256）：單向、不可還原 → 驗證完整性、存密碼
  • 加密：可用金鑰還原 → 保護機密性
  • 對應 CIA：雜湊↔完整性、加密↔機密性

🔑 密碼儲存

  • 要用慢雜湊＋加鹽（bcrypt/argon2）；絕不可明文或 md5
  • 系統只需「比對」不需還原 → 所以用雜湊不是加密

第1章 網路基礎

🔑 OSI 七層（上→下）＋對應攻擊

  • 7應用層 HTTP/DNS/SSH（SQLi/XSS）｜6表示層 加密/格式｜5會話層 建立/終止
  • 4傳輸層 TCP/UDP（SYN洪水）｜3網路層 IP/路由器（IP欺騙）｜2資料鏈路層 MAC/ARP/交換機（ARP欺騙）｜1實體層 位元流
  • ISO 1984 提出；TCP/IP 為 4-5 層實用模型

🔑 TCP vs UDP / IP

  • TCP：連接導向、可靠、有序、三向交握；UDP：無連接、快、無重傳
  • IP：無連接、盡力而為、不可靠；IPv4 32位／IPv6 128位

🔑 IP 位址

  • 私有(RFC1918)：10/8、172.16/12、192.168/16
  • 特殊：127.0.0.1本機｜169.254 APIPA(DHCP失敗)｜255.255.255.255廣播｜0.0.0.0/0預設路由

🔑 子網路（會算）

  • 網路位址 = IP AND 遮罩；可用主機 = 2^(主機位)−2（扣網路+廣播）
  • /24=255.255.255.0；/30 點對點(2台)；廣播=該段最後一個(主機位全1)
  • NAT：私→公、PAT多對一(靠埠)；VLAN(L2) vs 子網(L3)

🔑 通訊埠 & Wireshark

  • 0–1023 熟知｜1024–49151 註冊｜49152–65535 動態（總 0–65535）
  • HTTP80／HTTPS443／SSH22／DNS53／FTP21,20／SMTP25／RDP3389
  • 顯示過濾 ip.addr==x and tcp.port==443；掃埠工具 Nmap

第1章 網路服務（DNS/HTTP/SSH）

🔑 DNS

  • 用途：域名→IP（分散式電話簿）；遞迴(伺服器代查全) vs 迭代
  • 記錄：A(IPv4)｜AAAA(IPv6)｜CNAME(別名)｜MX(郵件)｜NS｜PTR(反查)
  • DNSSEC=驗證/完整性(不加密)；DoH(443)/DoT(853)=加密查詢

🔑 HTTP / 狀態碼

  • 方法：GET(參數在URL)／POST(body,送表單)／PUT／DELETE；無狀態→Cookie/Session
  • 2xx成功｜301永久/302臨時｜401需登入｜403沒權限｜404找不到｜500伺服器錯

🔑 HTTPS & Cookie 安全

  • HTTPS = HTTP + TLS(443)：機密性+身分驗證+完整性；憑證由 CA 簽發
  • Cookie：Secure(只走HTTPS,防竊聽)｜HttpOnly(防XSS竊取)｜SameSite(防CSRF)

FTP / SSH（明文↔加密成對）

  • FTP(21控制/20資料,明文)→ FTPS/SFTP(走SSH 22)
  • SSH(22,加密) vs Telnet(明文)；公鑰認證比密碼安全

🔑 Ping / ICMP / TTL

  • ping 靠 ICMP：送 Echo Request(Type8)、回 Echo Reply(Type0)；Type3不可達/Type11逾時
  • TTL 每經一路由器−1，歸零丟棄；預設 Linux64/Win128/Cisco254

第1章 威脅・偵查・分類

網路類型與指標

  • LAN(小快)/WAN(跨區,VPN)；有線較安全、無線用WPA3；P2P vs 主從
  • 指標：頻寬(最大速率)、延遲(時間)、吞吐量、抖動、遺失率（頻寬與延遲無關）

🔑 攻擊與防禦

  • DDoS→破壞可用性(CDN/清洗)；MITM→HTTPS/VPN/憑證；ARP欺騙→一IP多MAC
  • 資安設備：IDS(偵測告警) vs IPS(主動阻擋)；NAC、SIEM(集中日誌)

🔑 惡意程式種類

  • 蠕蟲(自我複製、網路自走)｜病毒(寄生宿主)｜木馬(偽裝)｜後門(潛伏通道)｜勒索(加密勒贖)

🔑 三種安全評估

  • 弱點掃描：自動找「已知漏洞」(Nessus/OpenVAS)、偏廣
  • 滲透測試：模擬真實攻擊、實際利用（黑/白/灰箱）、偏深；都需授權
  • 駭客入侵＝未授權；框架：Cyber Kill Chain、MITRE ATT&CK、CISSP 8領域

第2章 系統基礎

🔑 Linux 權限

  • r=4 w=2 x=1；755=rwxr-xr-x、644=rw-r--r--、敏感檔 600/700
  • 對象 u/g/o；chmod/chown/chgrp/umask；/etc/shadow=加密密碼只root可讀
  • 特殊權限：SUID(4)以擁有者執行(root+SUID易提權)｜SGID(2)｜Sticky(1,/tmp限刪)

常用指令 / Shell 安全

  • tail -f即時看日誌｜grep找可疑｜ps aux找程序｜ss/netstat看連線｜管道 |
  • rm -rf /破壞性→--preserve-root；PATH劫持=竄改PATH跑惡意程式

🔑 Windows 安全

  • UAC(提權前確認)｜Defender｜BitLocker(全碟) vs EFS(單檔)｜TPM/安全開機
  • 核心/使用者模式(權限隔離)；RDP要開 NLA；事件檢視器(攻擊者會清日誌)；Sysinternals

🔑 帳號與驗證

  • MFA 三因素：知識(密碼)/擁有(裝置)/生物(指紋)；要兩種不同類
  • 暴力破解(單帳號狂試) vs 密碼噴灑(一密碼試多帳號,避鎖定)；APT；SELinux/AppArmor

第3章 網站開發

HTML / CSS / JS

  • HTML結構(block獨佔行 vs inline並排；語意 nav/header/article)；表單 method GET/POST、label for↔id
  • CSS：選擇器 #id .class 元素；盒模型 margin(外)/padding(內)；Flexbox一維 vs Grid二維；@media=RWD；rgba 第4值=透明度
  • JS：let/const(區塊作用域) vs var；innerText安全 vs innerHTML會解析(XSS)

🔑 XSS 三型

  • 儲存型(存DB,危害最廣)｜反射型(URL夾帶+釣魚)｜DOM型(前端 Source→Sink)
  • 防禦：HTML實體編碼、用 textContent、CSP、HttpOnly Cookie

🔑 CSRF

  • 借「已登入身分」偽造請求(轉帳)；防：CSRF Token、SameSite、驗 Referer
  • 一句話：XSS=跑我的碼；CSRF=借你的身分

安全開發要點

  • 驗證與授權、輸入過濾(後端做,前端可繞過)、HTTPS、安全Cookie、防 SQLi/XSS/CSRF

第4章 Web 安全（OWASP/掃描/WAF）

⚠️ 本課 2025 版排序，與網路上 2021 版不同，以講義為準！

🔑 OWASP Top 10（2025）

A01存取控制失效A06不安全設計
A02安全配置錯誤A07認證失效
A03軟體供應鏈A08軟體/資料完整性失效
A04加密機制失效A09安全日誌/告警失效
A05注入 InjectionA10異常處理不當(fail-closed)
  • 防護：A01→最小權限/強制授權｜A04→HTTPS/強演算法｜A05→參數化查詢｜A07→MFA
  • 案例：Equifax(Struts未修補)／SolarWinds(供應鏈)／Capital One(WAF錯配+SSRF)／WannaCry(MS17-010 SMB)

🔑 漏洞掃描

  • 工具：OWASP ZAP、Nikto、OpenVAS、Burp、Nessus
  • CVSS 0–10（危急9-10/高7-8.9/中4-6.9/低）；誤報(多報假) vs 漏報(放過真,更危險)；DevSecOps

🔑 防火牆 vs WAF

  • 網路層防火牆(L3/L4,看IP/埠；封包過濾/狀態檢測/NGFW)
  • WAF(L7)：反向代理、深檢HTTP、防XSS/SQLi；虛擬補丁擋零日；ModSecurity/NAXSI

第5章 資料庫基礎

🔑 SQL 分類 DDL/DML/DCL

  • DDL定義結構(CREATE/ALTER)｜DML操作資料(SELECT/INSERT/UPDATE/DELETE)｜DCL控權限(GRANT/REVOKE)
  • RDBMS(MySQL,表格關聯) vs NoSQL(Not Only SQL,鍵值/文件/圖形)

查詢與子句

  • 順序：FROM→WHERE(過濾列)→GROUP BY→HAVING(過濾聚合)→SELECT→ORDER BY→LIMIT/OFFSET(分頁)
  • 聚合 COUNT/SUM/AVG/MIN/MAX；CASE WHEN；NULL用 IS NULL/COALESCE(不能=NULL)

🔑 關聯

  • 主鍵PK：唯一、不可重複/NULL、一表一個；外鍵FK ON DELETE RESTRICT(禁刪)/CASCADE(連刪)
  • INNER JOIN(交集) vs LEFT JOIN(左全+右NULL)；UNION(去重) vs UNION ALL(不去重)；子查詢

其他

  • 索引加速查找(EXPLAIN)；交易 BEGIN/COMMIT/ROLLBACK 保原子性(ACID的A)

第6章 資料庫安全

資料庫安全六面向

  • 存取控制(避sa/root)、資料加密(AES/RSA+金鑰管理)、備份、網路(SSL/防火牆)、應用(參數化)、監控稽核

🔑 SQL Injection 原理

  • 未過濾輸入被當 SQL：' OR '1'='1使條件恆真；admin'--用註解截斷密碼檢查
  • 類型：錯誤型(讀錯誤訊息)、布林盲注、時間盲注

🔑 union-based 完整攻擊鏈

  • order by N--+ 試欄位數 → union select database(),version() 探環境
  • 查 information_schema 列舉表/欄 → GROUP_CONCAT 把多筆壓成一格撈出帳密
  • 注入點閉合：試 '、')、')) 猜後端怎麼包參數

🔑 防禦（三層縱深）

  • 參數化查詢/Prepared Statement(語法層,主力)
  • 最小權限DB帳號(權限層,縮小爆炸半徑)；Stored Procedure 不等於安全(內部動態SQL仍可注入)
  • 不明文存密碼、關詳細錯誤、WAF；工具 sqlmap(自動化驗證)

管理與機密

  • GRANT/REVOKE授/收權；mysqldump備份(完整/差異/增量)
  • .env存帳密：放文件根目錄外、擋HTTP請求、加 .gitignore（勿把密碼推上 GitHub）
